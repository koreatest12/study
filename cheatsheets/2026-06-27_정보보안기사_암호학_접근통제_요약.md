# 정보보안기사 암호학·접근통제 핵심 요약

> 정보보안 일반 과목의 고빈도 개념을 정리한 암기용 자료입니다.

---

## 1. 정보보안 3요소

| 요소 | 의미 | 예시 |
|---|---|---|
| 기밀성 | 허가된 사용자만 정보 접근 | 암호화, 접근통제 |
| 무결성 | 정보가 임의로 변경되지 않음 | 해시, 전자서명 |
| 가용성 | 필요한 시점에 정상 사용 가능 | 이중화, 백업, DDoS 대응 |

---

## 2. 대칭키 암호와 공개키 암호

| 구분 | 대칭키 | 공개키 |
|---|---|---|
| 키 구조 | 같은 키로 암호화/복호화 | 공개키와 개인키 사용 |
| 속도 | 빠름 | 느림 |
| 장점 | 대용량 데이터 암호화에 적합 | 키 분배 문제 완화, 전자서명 가능 |
| 단점 | 키 분배가 어려움 | 연산 비용 큼 |
| 예시 | AES, SEED, ARIA, DES | RSA, ECC, ElGamal |

---

## 3. 해시 함수

| 항목 | 설명 |
|---|---|
| 목적 | 무결성 검증 |
| 특징 | 일방향성, 고정 길이 출력, 충돌 저항성 |
| 예시 | SHA-256, SHA-3 |
| 사용 예 | 파일 무결성 검증, 비밀번호 저장, 전자서명 전 메시지 요약 |

주의할 점:

- 해시는 암호화가 아니다.
- 해시는 복호화 개념이 없다.
- 비밀번호 저장 시 단순 해시보다 Salt와 반복 연산이 필요하다.

---

## 4. 전자서명

| 단계 | 내용 |
|---|---|
| 서명 생성 | 송신자가 자신의 개인키로 메시지 해시값에 서명 |
| 서명 검증 | 수신자가 송신자의 공개키로 검증 |
| 제공 속성 | 무결성, 인증, 부인방지 |

암기 문장:

> 전자서명은 개인키로 서명하고 공개키로 검증한다.

## 5. PKI 구성요소

| 구성요소 | 역할 |
|---|---|
| CA | 인증서 발급 및 관리 |
| RA | 인증서 신청자의 신원 확인 |
| 인증서 | 공개키와 소유자 정보를 묶은 전자문서 |
| CRL | 폐지된 인증서 목록 |
| OCSP | 인증서 상태를 실시간 확인 |

---

## 6. 접근통제 모델

| 모델 | 설명 | 핵심 |
|---|---|---|
| DAC | 소유자가 접근 권한을 결정 | 임의적 접근통제 |
| MAC | 보안 등급과 정책에 따라 접근 결정 | 강제적 접근통제 |
| RBAC | 역할에 따라 권한 부여 | 직무 기반 권한 관리 |

---

## 7. 보안 모델

| 모델 | 보호 목표 | 대표 규칙 |
|---|---|---|
| Bell-LaPadula | 기밀성 | No Read Up, No Write Down |
| Biba | 무결성 | No Read Down, No Write Up |
| Clark-Wilson | 상업적 무결성 | 잘 형성된 트랜잭션, 직무 분리 |

## 8. 예상문제

### 문제 1
전자서명에서 서명 검증에 사용하는 키는?

1. 송신자의 개인키  
2. 송신자의 공개키  
3. 수신자의 개인키  
4. 세션키

**정답: 2**

**해설:** 전자서명은 송신자의 개인키로 생성하고 송신자의 공개키로 검증한다.

### 문제 2
Bell-LaPadula 모델의 핵심 보호 대상은?

1. 기밀성  
2. 무결성  
3. 가용성  
4. 책임추적성

**정답: 1**

**해설:** Bell-LaPadula 모델은 기밀성 보호를 목적으로 한다.

### 문제 3
해시 함수의 특징으로 적절하지 않은 것은?

1. 고정 길이 출력  
2. 일방향성  
3. 복호화 가능  
4. 무결성 검증에 사용

**정답: 3**

**해설:** 해시는 암호화가 아니며 복호화 개념이 없다.
