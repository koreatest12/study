# 정보보안기사 웹 보안 취약점 요약

> 어플리케이션 보안 과목의 핵심 취약점과 방어 방법을 정리한 자료입니다.

---

## 1. SQL Injection

| 항목 | 내용 |
|---|---|
| 개념 | 사용자 입력값을 이용해 SQL 구문을 조작하는 공격 |
| 영향 | 인증 우회, 정보 유출, 데이터 변조/삭제 |
| 주요 유형 | Error Based, Union Based, Blind SQL Injection |
| 대응 | Prepared Statement, 입력값 검증, DB 권한 최소화, 오류 메시지 제한 |

### 핵심 암기

> SQL Injection의 가장 대표적인 대응은 Prepared Statement이다.

---

## 2. XSS

| 항목 | 내용 |
|---|---|
| 개념 | 웹 페이지에 악성 스크립트를 삽입해 사용자의 브라우저에서 실행시키는 공격 |
| 영향 | 쿠키 탈취, 세션 탈취, 피싱, 악성 페이지 이동 |
| 대응 | 출력값 인코딩, CSP, 입력값 검증, HttpOnly Cookie |

### XSS 유형

| 유형 | 설명 |
|---|---|
| Stored XSS | 악성 스크립트가 서버에 저장됨 |
| Reflected XSS | 요청값이 응답에 반사되어 실행됨 |
| DOM XSS | 브라우저 DOM 처리 과정에서 실행됨 |

---

## 3. CSRF

| 항목 | 내용 |
|---|---|
| 개념 | 사용자의 인증 상태를 악용해 원치 않는 요청을 전송하게 하는 공격 |
| 영향 | 비밀번호 변경, 게시글 작성, 결제 요청 등 비정상 행위 |
| 대응 | CSRF Token, SameSite Cookie, Referer 검증, 중요 작업 재인증 |

### XSS와 CSRF 차이

| 구분 | XSS | CSRF |
|---|---|---|
| 공격 위치 | 사용자 브라우저에서 스크립트 실행 | 사용자의 인증 상태 악용 |
| 핵심 피해 | 세션 탈취, 스크립트 실행 | 원치 않는 요청 수행 |
| 주요 대응 | 출력값 인코딩 | CSRF Token |

---

## 4. 파일 업로드 취약점

| 항목 | 내용 |
|---|---|
| 개념 | 악성 파일을 서버에 업로드하여 실행시키는 취약점 |
| 대표 피해 | 웹쉘 업로드, 서버 명령 실행, 내부망 침투 |
| 대응 | 확장자 검증, MIME 검증, 파일명 변경, 저장 경로 분리, 실행 권한 제거 |

---

## 5. 세션 보안

| 취약점 | 설명 | 대응 |
|---|---|---|
| 세션 하이재킹 | 세션 ID 탈취 후 사용자로 위장 | HTTPS, Secure Cookie, HttpOnly |
| 세션 고정 | 공격자가 지정한 세션 ID를 피해자에게 사용하게 함 | 로그인 후 세션 ID 재발급 |
| 세션 예측 | 약한 세션 ID 생성 규칙 악용 | 충분한 난수성 확보 |

## 6. 예상문제

### 문제 1
SQL Injection 방어에 가장 효과적인 방법은?

1. Prepared Statement 사용  
2. 오류 메시지 상세 출력  
3. 관리자 권한 DB 계정 사용  
4. 사용자 입력값을 SQL 문자열에 직접 연결

**정답: 1**

**해설:** Prepared Statement는 SQL 구조와 입력값을 분리하여 SQL 구문 조작을 방지한다.

### 문제 2
XSS 방어의 핵심 조치는?

1. 출력값 인코딩  
2. 포트 차단  
3. 세션 시간 무제한 설정  
4. DB 관리자 권한 부여

**정답: 1**

**해설:** XSS는 브라우저에서 스크립트가 실행되는 공격이므로 출력값 인코딩이 핵심이다.

### 문제 3
로그인 후 세션 ID를 새로 발급하는 이유는?

1. 세션 고정 공격 방지  
2. DNS Spoofing 방지  
3. ARP Spoofing 방지  
4. DDoS 방지

**정답: 1**

**해설:** 세션 고정 공격은 공격자가 미리 정한 세션 ID를 피해자에게 사용하게 하는 공격이다. 로그인 후 세션 ID를 재발급하면 방어할 수 있다.
