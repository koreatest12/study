# 1과목. 시스템 보안

## 핵심 개념

시스템 보안은 운영체제, 서버, 계정, 권한, 로그, 백업, 악성코드, 취약점 관리 등을 통해 시스템을 안전하게 운영하는 과목이다.

## 주요 개념 정리

| 구분 | 핵심 내용 |
|---|---|
| 운영체제 보안 | 계정 관리, 권한 관리, 파일 시스템 보안, 프로세스 관리 |
| 윈도우 보안 | 이벤트 로그, 레지스트리, NTFS 권한, 계정 정책 |
| 리눅스/유닉스 보안 | /etc/passwd, /etc/shadow, chmod, chown, umask, su, sudo |
| 접근통제 | DAC, MAC, RBAC, 최소 권한 원칙 |
| 로그 관리 | syslog, auth.log, secure, wtmp, utmp, Windows Event Viewer |
| 악성코드 | 바이러스, 웜, 트로이목마, 랜섬웨어, 스파이웨어 |
| 백업과 복구 | 전체 백업, 증분 백업, 차등 백업, 복구 절차 |
| 취약점 관리 | 패치 관리, 불필요 서비스 제거, 포트 점검, 보안 설정 점검 |

## 암기 포인트

| 항목 | 설명 |
|---|---|
| SetUID | 실행 파일 소유자의 권한으로 실행 |
| SetGID | 실행 파일 그룹의 권한으로 실행 |
| Sticky Bit | 공용 디렉터리에서 파일 소유자만 삭제 가능 |
| umask | 새 파일/디렉터리 생성 시 기본 권한 제한값 |
| 최소 권한 원칙 | 업무 수행에 필요한 최소 권한만 부여 |
| 계정 잠금 정책 | 반복 로그인 실패 시 계정을 잠가 무차별 대입 공격을 완화 |

## 예상문제

### 문제 1
리눅스에서 `/tmp`와 같은 공용 디렉터리에서 다른 사용자의 파일 삭제를 제한하는 특수 권한은?

1. SetUID  
2. SetGID  
3. Sticky Bit  
4. umask

**정답: 3**

**해설:** Sticky Bit는 공용 디렉터리에서 파일 소유자 또는 관리자만 파일을 삭제할 수 있도록 제한한다. `/tmp` 디렉터리에 자주 적용된다.

### 문제 2
최소 권한 원칙에 대한 설명으로 가장 적절한 것은?

1. 모든 사용자에게 관리자 권한을 부여한다.  
2. 업무 수행에 필요한 최소 권한만 부여한다.  
3. 로그 기록을 비활성화한다.  
4. 공용 계정을 사용한다.

**정답: 2**

**해설:** 최소 권한 원칙은 권한 오남용과 침해 사고 피해 범위를 줄이기 위해 반드시 필요한 권한만 부여하는 원칙이다.

### 문제 3
마지막 전체 백업 이후 변경된 데이터를 계속 누적하여 백업하는 방식은?

1. 전체 백업  
2. 증분 백업  
3. 차등 백업  
4. 미러링

**정답: 3**

**해설:** 차등 백업은 마지막 전체 백업 이후 변경된 모든 데이터를 백업한다. 증분 백업은 마지막 백업 이후 변경분만 백업한다.

# 2과목. 네트워크 보안

## 핵심 개념

네트워크 보안은 TCP/IP, OSI 7계층, 라우팅, 포트, 방화벽, IDS/IPS, VPN, 무선 보안, 공격 기법을 다룬다.

## 주요 개념 정리

| 구분 | 핵심 내용 |
|---|---|
| OSI 7계층 | 물리, 데이터링크, 네트워크, 전송, 세션, 표현, 응용 |
| TCP/IP | IP, TCP, UDP, ICMP, ARP, DNS, HTTP, FTP |
| 주요 포트 | SSH 22, DNS 53, HTTP 80, HTTPS 443, SMTP 25 |
| 방화벽 | 패킷 필터링, 상태 기반 방화벽, 프록시 방화벽 |
| IDS/IPS | 침입 탐지와 침입 차단 |
| VPN | IPsec, SSL VPN, 터널링, NAT-T |
| 공격 기법 | 스푸핑, 스니핑, 세션 하이재킹, DoS/DDoS |
| 무선 보안 | WPA2, WPA3, 802.1X, 인증 및 암호화 |

## 암기 포인트

### TCP 3-way handshake

1. Client → Server: SYN  
2. Server → Client: SYN/ACK  
3. Client → Server: ACK

### 주요 포트

| 서비스 | 포트 |
|---|---|
| SSH | TCP 22 |
| Telnet | TCP 23 |
| SMTP | TCP 25 |
| DNS | UDP/TCP 53 |
| HTTP | TCP 80 |
| HTTPS | TCP 443 |
| IKE | UDP 500 |
| NAT-T | UDP 4500 |
| ESP | IP Protocol 50 |
| AH | IP Protocol 51 |

### IDS와 IPS 비교

| 구분 | IDS | IPS |
|---|---|---|
| 의미 | 침입 탐지 시스템 | 침입 방지 시스템 |
| 역할 | 이상 행위 탐지 및 경고 | 탐지 후 차단 |
| 위치 | 미러링 구간 | 네트워크 경로상 인라인 |
| 핵심 | 탐지 | 탐지 + 차단 |

## 예상문제

### 문제 1
TCP 3-way handshake의 올바른 순서는?

1. SYN → ACK → FIN  
2. SYN → SYN/ACK → ACK  
3. ACK → SYN → SYN/ACK  
4. SYN/ACK → SYN → ACK

**정답: 2**

**해설:** TCP 연결 수립은 SYN, SYN/ACK, ACK 순서로 진행된다. FIN은 연결 종료 시 사용된다.

### 문제 2
침입 탐지뿐 아니라 차단 기능까지 수행하는 보안 장비는?

1. IDS  
2. IPS  
3. Hub  
4. Repeater

**정답: 2**

**해설:** IDS는 탐지와 경고가 중심이고, IPS는 탐지 후 패킷 차단까지 수행한다.

### 문제 3
NAT 환경에서 IPsec ESP 패킷을 UDP로 캡슐화하여 전송하는 기술은?

1. SSH  
2. NAT-T  
3. ARP  
4. GRE

**정답: 2**

**해설:** NAT-T는 NAT Traversal의 약자로, IPsec ESP 패킷을 UDP 4500으로 캡슐화하여 NAT 환경에서도 IPsec 통신이 가능하게 한다.

# 3과목. 어플리케이션 보안

## 핵심 개념

어플리케이션 보안은 웹 취약점, 데이터베이스 보안, 소프트웨어 개발 보안, API 보안, 세션 관리 등을 다룬다.

## 주요 개념 정리

| 구분 | 핵심 내용 |
|---|---|
| 웹 보안 | SQL Injection, XSS, CSRF, 파일 업로드 취약점 |
| 인증 보안 | 비밀번호 저장, 세션 고정, 세션 하이재킹 |
| 입력값 검증 | 화이트리스트 검증, 특수문자 필터링 |
| 출력값 인코딩 | XSS 방어 핵심 |
| DB 보안 | 권한 분리, 암호화, 감사 로그 |
| API 보안 | 인증 토큰, 권한 검증, CORS, Rate Limiting |
| 시큐어 코딩 | 예외 처리, 오류 메시지 제한, 안전한 함수 사용 |

## 암기 포인트

| 공격/취약점 | 대응 방법 |
|---|---|
| SQL Injection | Prepared Statement, 입력값 검증, DB 권한 최소화 |
| XSS | 출력값 인코딩, CSP, 입력값 검증 |
| CSRF | CSRF Token, SameSite Cookie, Referer 검증 |
| 파일 업로드 취약점 | 확장자/MIME 검증, 실행 권한 제거, 저장 경로 분리 |
| 세션 하이재킹 | HTTPS, 세션 재발급, HttpOnly/Secure Cookie |

## XSS 종류

| 종류 | 설명 |
|---|---|
| Stored XSS | 악성 스크립트가 서버에 저장됨 |
| Reflected XSS | 요청값이 응답에 반사되어 실행됨 |
| DOM XSS | 브라우저 DOM 처리 과정에서 실행됨 |

## 예상문제

### 문제 1
SQL Injection 공격에 대한 가장 적절한 대응 방법은?

1. Prepared Statement 사용  
2. 오류 메시지 상세 공개  
3. 사용자 입력을 그대로 SQL에 연결  
4. 관리자 권한 기본 부여

**정답: 1**

**해설:** Prepared Statement는 SQL 구문과 입력값을 분리하여 공격자가 입력값으로 SQL 구조를 변경하지 못하게 한다.

### 문제 2
사용자 입력이 서버 응답에 즉시 반영되어 실행되는 XSS 유형은?

1. Stored XSS  
2. DOM 기반 XSS  
3. Reflected XSS  
4. CSRF

**정답: 3**

**해설:** Reflected XSS는 사용자 입력값이 서버 응답에 그대로 반사되어 브라우저에서 실행되는 공격이다.

### 문제 3
CSRF 공격을 방어하는 대표적인 방법은?

1. 긴 URL 사용  
2. CSRF Token 검증  
3. 디버그 모드 활성화  
4. 공개 쿠키 사용

**정답: 2**

**해설:** CSRF Token은 요청이 정상 페이지에서 생성된 것인지 검증하는 값이다.

# 4과목. 정보보안 일반

## 핵심 개념

정보보안 일반은 보안의 기본 원칙, 암호학, 해시, 전자서명, 인증, 접근통제 모델, 보안 이론을 다룬다.

## 주요 개념 정리

| 구분 | 핵심 내용 |
|---|---|
| 보안 3요소 | 기밀성, 무결성, 가용성 |
| 추가 보안 요소 | 인증, 책임추적성, 부인방지 |
| 대칭키 암호 | AES, SEED, ARIA, DES |
| 공개키 암호 | RSA, ECC |
| 해시 함수 | SHA-256, SHA-3 |
| 전자서명 | 무결성, 인증, 부인방지 |
| PKI | 인증서, CA, RA, CRL, OCSP |
| 접근통제 모델 | DAC, MAC, RBAC |
| 보안 모델 | Bell-LaPadula, Biba, Clark-Wilson |

## 암기 포인트

### CIA

| 요소 | 의미 |
|---|---|
| 기밀성 | 허가된 사용자만 정보 접근 |
| 무결성 | 정보가 임의로 변경되지 않음 |
| 가용성 | 필요한 시점에 정상 사용 가능 |

### 대칭키와 공개키

| 구분 | 대칭키 | 공개키 |
|---|---|---|
| 키 개수 | 하나의 비밀키 | 공개키 + 개인키 |
| 속도 | 빠름 | 느림 |
| 용도 | 대량 데이터 암호화 | 키 교환, 전자서명 |
| 예시 | AES, SEED, ARIA | RSA, ECC |

### 보안 모델

| 모델 | 핵심 |
|---|---|
| Bell-LaPadula | 기밀성 중심, No Read Up / No Write Down |
| Biba | 무결성 중심, No Read Down / No Write Up |
| Clark-Wilson | 상업적 무결성, 트랜잭션 중심 |

## 예상문제

### 문제 1
정보보안의 3요소에 해당하지 않는 것은?

1. 기밀성  
2. 무결성  
3. 가용성  
4. 효율성

**정답: 4**

**해설:** 정보보안의 3요소는 기밀성, 무결성, 가용성이다.

### 문제 2
해시 함수의 주된 목적은?

1. 기밀성 제공  
2. 무결성 검증  
3. 부팅 속도 향상  
4. 키 길이 축소

**정답: 2**

**해설:** 해시 함수는 데이터가 변경되었는지 확인하는 무결성 검증에 주로 사용된다.

### 문제 3
전자서명 검증에 사용하는 키는?

1. 수신자의 개인키  
2. 송신자의 공개키  
3. 공용 비밀번호

**정답: 2**

**해설:** 전자서명은 송신자의 개인키로 생성하고 송신자의 공개키로 검증한다.

# 5과목. 정보보안 관리 및 법규

## 핵심 개념

정보보안 관리 및 법규는 보안 관리체계, 위험관리, 개인정보보호, 침해사고 대응, BCP/DRP, 관련 법규를 다룬다.

## 주요 개념 정리

| 구분 | 핵심 내용 |
|---|---|
| 정보보호 관리체계 | ISMS, ISMS-P, 정책, 조직, 책임과 역할 |
| 위험관리 | 자산, 위협, 취약점, 위험 분석과 처리 |
| 보호대책 | 관리적, 물리적, 기술적 보호조치 |
| 침해사고 대응 | 준비, 탐지, 분석, 대응, 복구, 보고 |
| 업무 연속성 | BCP, DRP, RTO, RPO |
| 개인정보보호 | 수집, 이용, 보관, 파기, 제3자 제공 |
| 주요 법규 | 개인정보보호법, 정보통신망법, 전자서명법 |

## 암기 포인트

### PDCA

| 단계 | 의미 |
|---|---|
| Plan | 계획 수립 |
| Do | 실행 |
| Check | 점검 |
| Act | 개선 |

### 위험관리 구성요소

| 요소 | 설명 |
|---|---|
| 자산 | 보호해야 할 대상 |
| 위협 | 자산에 피해를 줄 수 있는 원인 |
| 취약점 | 위협이 악용할 수 있는 약점 |
| 위험 | 위협이 취약점을 이용해 자산에 피해를 줄 가능성 |

### 보호조치 분류

| 구분 | 예시 |
|---|---|
| 관리적 보호조치 | 정책, 절차, 교육, 권한 관리 |
| 물리적 보호조치 | 출입통제, CCTV, 잠금장치 |
| 기술적 보호조치 | 암호화, 방화벽, 접근통제, 로그 |

## 예상문제

### 문제 1
ISMS 운영의 지속적 개선에 사용되는 순환 모델은?

1. SWOT  
2. PDCA  
3. FIFO  
4. RAID

**정답: 2**

**해설:** 정보보호 관리체계는 Plan-Do-Check-Act 사이클을 통해 지속적으로 개선한다.

### 문제 2
위험관리에서 위험을 구성하는 요소로 가장 적절한 것은?

1. 자산, 위협, 취약점  
2. CPU, 메모리, 디스크  
3. 사용자, 브라우저, 프린터  
4. 시간, 비용, 거리

**정답: 1**

**해설:** 위험은 보호 대상인 자산, 피해 원인인 위협, 위협이 악용할 수 있는 취약점을 기준으로 평가한다.

### 문제 3
개인정보는 목적 달성 후 원칙적으로 어떻게 처리해야 하는가?

1. 무기한 보관  
2. 외부 공개  
3. 지체 없이 파기  
4. 임의 제3자 제공

**정답: 3**

**해설:** 개인정보는 수집·이용 목적 달성 후 관련 법령상 보존 의무가 있는 경우를 제외하고 지체 없이 파기해야 한다.

# 최종 암기표

| 개념 | 반드시 기억할 내용 |
|---|---|
| CIA | 기밀성, 무결성, 가용성 |
| 최소 권한 | 필요한 권한만 부여 |
| SetUID | 파일 소유자 권한으로 실행 |
| Sticky Bit | 공용 디렉터리 삭제 제한 |
| IDS | 침입 탐지 |
| IPS | 침입 탐지 + 차단 |
| NAT-T | UDP 4500 |
| HTTPS | TCP 443 |
| DNS | UDP/TCP 53 |
| SQL Injection 방어 | Prepared Statement |
| XSS 방어 | 출력값 인코딩 |
| CSRF 방어 | CSRF Token |
| 해시 | 무결성 검증 |
| 전자서명 | 개인키 서명, 공개키 검증 |
| Bell-LaPadula | 기밀성 모델 |
| Biba | 무결성 모델 |
| PDCA | Plan → Do → Check → Act |
| 위험관리 | 자산 + 위협 + 취약점 |
| BCP | 업무 연속성 계획 |
| DRP | 재해 복구 계획 |

# 공부 순서 추천

1. 정보보안 일반: CIA, 암호, 인증, 접근통제 기초 정리
2. 시스템 보안: 리눅스/윈도우 계정, 권한, 로그, 백업 암기
3. 네트워크 보안: OSI 7계층, TCP/IP, 포트, 방화벽, IDS/IPS 정리
4. 어플리케이션 보안: SQL Injection, XSS, CSRF 중심 문제 풀이
5. 관리 및 법규: ISMS, 위험관리, 개인정보보호, 침해사고 대응 절차 암기
