# 3과목 어플리케이션 보안 심화 정리

## 1. 핵심 범위

어플리케이션 보안은 웹 취약점, 데이터베이스 보안, 인증/세션 관리, API 보안, 시큐어 코딩을 다룬다.

---

## 2. 웹 취약점 비교

| 취약점 | 핵심 | 대표 대응 |
|---|---|---|
| SQL Injection | SQL 구문 조작 | Prepared Statement |
| XSS | 브라우저 스크립트 실행 | 출력값 인코딩 |
| CSRF | 인증 상태 악용 | CSRF Token |
| 파일 업로드 | 웹쉘 업로드 | 실행 권한 제거 |
| 경로 조작 | ../로 파일 접근 | 경로 정규화 및 검증 |
| 인증 우회 | 권한 검증 누락 | 서버 측 권한 검증 |

---

## 3. SQL Injection 유형

| 유형 | 설명 |
|---|---|
| Error Based | DB 오류 메시지 활용 |
| Union Based | UNION으로 데이터 결합 |
| Blind | 참/거짓 또는 시간 지연으로 추론 |

---

## 4. XSS 유형

| 유형 | 설명 |
|---|---|
| Stored XSS | 서버에 저장되어 지속 실행 |
| Reflected XSS | 요청값이 응답에 반사 |
| DOM XSS | 클라이언트 DOM 처리 중 실행 |

---

## 5. 예상문제

### 문제 1
Blind SQL Injection의 특징은?

1. DB 오류 메시지만 이용한다.
2. 화면에 직접 출력되는 데이터만 이용한다.
3. 참/거짓 또는 시간 지연으로 데이터를 추론한다.
4. 네트워크 포트만 스캔한다.

**정답: 3**

**해설:** Blind SQL Injection은 직접적인 결과가 보이지 않아도 조건 반응을 통해 정보를 추론한다.

### 문제 2
CSRF Token의 목적은?

1. 요청의 정당성 검증
2. 포트 암호화
3. DB 인덱싱
4. DNS 캐싱

**정답: 1**

**해설:** CSRF Token은 정상 페이지에서 생성된 요청인지 검증하는 데 사용된다.
