## 1과목. 시스템 보안

### 1. Sticky Bit의 설명으로 옳은 것은?
1. 파일 소유자 권한으로 실행한다.
2. 그룹 권한으로 실행한다.
3. 공용 디렉터리에서 파일 소유자만 삭제할 수 있게 한다.
4. 새 파일의 기본 권한을 설정한다.

**정답: 3**  
**해설:** Sticky Bit는 `/tmp` 같은 공용 디렉터리에서 파일 삭제 권한을 제한한다.

### 2. `/etc/shadow` 파일의 역할은?
1. 사용자 계정 기본 정보 저장
2. 암호화된 패스워드 해시 저장
3. DNS 서버 정보 저장
4. 서비스 포트 정보 저장

**정답: 2**  
**해설:** `/etc/shadow`는 패스워드 해시와 계정 만료 정책을 저장한다.

### 3. umask의 역할은?
1. 파일 소유자 변경
2. 기본 권한 제한
3. 관리자 권한 실행
4. 시스템 종료

**정답: 2**  
**해설:** umask는 새 파일/디렉터리 생성 시 부여될 권한에서 제거할 권한을 지정한다.

### 4. 최소 권한 원칙의 목적은?
1. 모든 권한 부여
2. 편의성 극대화
3. 필요한 권한만 부여하여 위험 감소
4. 로그 삭제 자동화

**정답: 3**  
**해설:** 최소 권한 원칙은 권한 오남용과 피해 범위를 줄인다.

### 5. 윈도우 보안 이벤트 확인 도구는?
1. Event Viewer
2. cron
3. chmod
4. iptables

**정답: 1**  
**해설:** Windows Event Viewer는 보안, 시스템, 응용 프로그램 로그를 확인한다.

## 2과목. 네트워크 보안

### 6. TCP 3-way handshake 순서는?
1. SYN → SYN/ACK → ACK
2. ACK → SYN → FIN
3. FIN → ACK → SYN
4. PSH → ACK → FIN

**정답: 1**  
**해설:** TCP 연결 수립은 SYN, SYN/ACK, ACK 순서다.

### 7. HTTPS 기본 포트는?
1. 22
2. 53
3. 80
4. 443

**정답: 4**  
**해설:** HTTPS는 TCP 443 포트를 사용한다.

### 8. DNS 기본 포트는?
1. 21
2. 22
3. 53
4. 110

**정답: 3**  
**해설:** DNS는 UDP/TCP 53을 사용한다.

### 9. IDS와 IPS의 차이로 옳은 것은?
1. IDS는 차단, IPS는 탐지만 수행한다.
2. IDS는 탐지 중심, IPS는 탐지와 차단을 수행한다.
3. 둘 다 물리계층 장비다.
4. 둘 다 암호 알고리즘이다.

**정답: 2**  
**해설:** IDS는 탐지와 경고, IPS는 탐지 후 차단까지 수행한다.

### 10. NAT-T의 포트는?
1. UDP 500
2. UDP 4500
3. TCP 443
4. TCP 3389

**정답: 2**  
**해설:** NAT-T는 UDP 4500을 사용한다.

## 3과목. 어플리케이션 보안

### 11. SQL Injection 방어의 핵심은?
1. Prepared Statement
2. 상세 오류 메시지 출력
3. 관리자 DB 계정 사용
4. 입력값 직접 연결

**정답: 1**  
**해설:** Prepared Statement는 SQL 구조와 사용자 입력을 분리한다.

### 12. XSS 방어의 핵심은?
1. 출력값 인코딩
2. 포트 차단
3. 라우팅 변경
4. 디스크 포맷

**정답: 1**  
**해설:** XSS는 스크립트 실행 공격이므로 출력값 인코딩이 중요하다.

### 13. CSRF의 특징은?
1. 사용자의 인증 상태를 악용한다.
2. 패킷을 증폭한다.
3. MAC 주소를 위조한다.
4. 파일 권한을 변경한다.

**정답: 1**  
**해설:** CSRF는 로그인된 사용자의 권한으로 원치 않는 요청을 수행하게 한다.

### 14. 파일 업로드 취약점 대응으로 부적절한 것은?
1. 확장자 검증
2. MIME 검증
3. 실행 권한 제거
4. 웹 루트에서 바로 실행 허용

**정답: 4**  
**해설:** 웹 루트 실행 허용은 웹쉘 공격으로 이어질 수 있다.

### 15. 세션 고정 공격 대응은?
1. 로그인 후 세션 ID 재발급
2. 세션 시간 무제한
3. 쿠키 평문 저장
4. 모든 사용자 같은 세션 사용

**정답: 1**  
**해설:** 로그인 후 세션 ID를 재발급하면 세션 고정 공격을 완화할 수 있다.

## 5과목. 관리 및 법규

### 22. RTO의 의미는?
1. 복구 목표 시간

3. 암호 키 길이
4. 패킷 크기

**정답: 1**
