# 1과목 시스템 보안 심화 정리

## 1. 핵심 범위

시스템 보안은 운영체제, 계정, 권한, 로그, 백업, 악성코드, 서버 보안 설정을 다룬다.

---

## 2. 리눅스 계정 파일

| 파일 | 설명 |
|---|---|
| /etc/passwd | 사용자 계정 기본 정보 |
| /etc/shadow | 패스워드 해시와 만료 정책 |
| /etc/group | 그룹 정보 |
| /etc/sudoers | sudo 권한 설정 |

---

## 3. 권한 관리

| 명령어 | 역할 |
|---|---|
| chmod | 파일 권한 변경 |
| chown | 소유자 변경 |
| chgrp | 그룹 변경 |
| umask | 기본 권한 제한 |

---

## 4. 특수 권한

| 권한 | 설명 | 예시 |
|---|---|---|
| SetUID | 파일 소유자 권한으로 실행 | passwd 명령 |
| SetGID | 파일 그룹 권한으로 실행 | 공유 디렉터리 |
| Sticky Bit | 소유자만 삭제 가능 | /tmp |

---

## 5. 로그 분석

| 로그 | 설명 |
|---|---|
| /var/log/messages | 시스템 일반 로그 |
| /var/log/secure | 인증 및 보안 관련 로그 |
| /var/log/auth.log | Debian 계열 인증 로그 |
| wtmp | 로그인 성공 기록 |
| btmp | 로그인 실패 기록 |
| utmp | 현재 로그인 사용자 |

---

## 6. 예상문제

### 문제 1
SetUID가 설정된 실행 파일의 특징은?

1. 실행한 사용자의 권한으로만 실행된다.
2. 파일 소유자의 권한으로 실행된다.
3. 파일 그룹을 삭제한다.
4. 네트워크 포트를 차단한다.

**정답: 2**

**해설:** SetUID는 실행 파일 소유자의 권한으로 프로그램이 실행되게 한다.

### 문제 2
로그인 실패 기록을 확인할 때 주로 참고하는 파일은?

1. wtmp
2. btmp
3. utmp
4. hosts

**정답: 2**

**해설:** btmp는 로그인 실패 기록을 저장한다.
