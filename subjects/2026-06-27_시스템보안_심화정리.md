# 시스템 보안 심화정리

---

## 1. 계정 보안

| 항목 | 핵심 내용 |
|---|---|
| 계정 잠금 | 반복 로그인 실패 시 계정 잠금 |
| 패스워드 정책 | 길이, 복잡도, 변경 주기, 재사용 제한 |
| 공용 계정 금지 | 책임추적성 확보를 위해 개인별 계정 사용 |
| 휴면 계정 관리 | 장기 미사용 계정 비활성화 또는 삭제 |

---

## 2. 리눅스 주요 파일

| 파일 | 설명 |
|---|---|
| /etc/passwd | 사용자 계정 기본 정보 |
| /etc/shadow | 패스워드 해시와 만료 정보 |
| /etc/group | 그룹 정보 |
| /etc/sudoers | sudo 권한 설정 |
| /var/log/secure | 인증 관련 로그 |
| /var/log/messages | 시스템 일반 로그 |

---

## 3. 권한 관리

| 권한 | 숫자 | 의미 |
|---|---:|---|
| read | 4 | 읽기 |
| write | 2 | 쓰기 |
| execute | 1 | 실행 |

예시:

- 755: 소유자 읽기/쓰기/실행, 그룹 읽기/실행, 기타 읽기/실행
- 644: 소유자 읽기/쓰기, 그룹 읽기, 기타 읽기
- 600: 소유자만 읽기/쓰기

---

## 4. 특수 권한

| 권한 | 설명 |
|---|---|
| SetUID | 실행 파일 소유자 권한으로 실행 |
| SetGID | 실행 파일 그룹 권한으로 실행 |
| Sticky Bit | 공용 디렉터리에서 파일 소유자만 삭제 가능 |

---

## 5. 로그 분석 포인트

| 로그 | 확인 내용 |
|---|---|
| 로그인 실패 | 무차별 대입 공격 가능성 |
| 권한 상승 | sudo, su 사용 이력 |
| 서비스 오류 | 장애 또는 침해 징후 |
| 파일 변경 | 중요 설정 파일 변조 여부 |

---

## 6. 예상문제

### 문제 1
책임추적성을 확보하기 위한 계정 관리 원칙은?

1. 공용 계정 사용
2. 개인별 계정 사용
3. 모든 계정 관리자 권한 부여
4. 로그 비활성화

정답: 2

해설: 개인별 계정을 사용해야 행위자를 추적할 수 있다.

### 문제 2
sudo 권한 설정 파일은?

1. /etc/passwd
2. /etc/shadow
3. /etc/sudoers
4. /etc/hosts

정답: 3

해설: /etc/sudoers에서 sudo 권한을 설정한다.
