# 정보보안 일반 심화정리

---

## 1. CIA와 AAA

| 구분 | 의미 |
|---|---|
| 기밀성 | 허가된 사용자만 정보에 접근 |
| 무결성 | 정보가 임의로 변경되지 않음 |
| 가용성 | 필요한 시점에 정상 사용 가능 |
| 인증 | 사용자가 누구인지 확인 |
| 인가 | 인증된 사용자에게 권한 부여 |
| 책임추적성 | 행위자를 추적 가능하게 함 |

---

## 2. 암호 기술

| 구분 | 설명 | 예시 |
|---|---|---|
| 대칭키 | 같은 키로 암호화와 복호화 | AES, SEED, ARIA |
| 공개키 | 공개키와 개인키 사용 | RSA, ECC |
| 해시 | 고정 길이 요약값 생성 | SHA-256, SHA-3 |
| 전자서명 | 개인키 서명, 공개키 검증 | RSA 서명, ECDSA |

---

## 3. 접근통제

| 모델 | 핵심 |
|---|---|
| DAC | 소유자가 권한을 결정 |
| MAC | 보안 등급과 정책으로 강제 통제 |
| RBAC | 역할에 따라 권한 부여 |

---

## 4. 보안 모델

| 모델 | 목표 | 암기 포인트 |
|---|---|---|
| Bell-LaPadula | 기밀성 | No Read Up, No Write Down |
| Biba | 무결성 | No Read Down, No Write Up |
| Clark-Wilson | 상업적 무결성 | 직무 분리와 트랜잭션 |

---

## 5. 예상문제

### 문제 1
해시 함수의 주된 목적은?

1. 무결성 검증
2. 네트워크 라우팅
3. 디스크 포맷
4. 계정 생성

정답: 1

해설: 해시는 데이터 변경 여부를 확인하는 무결성 검증에 사용된다.

### 문제 2
역할 기반 접근통제 모델은?

1. DAC
2. MAC
3. RBAC
4. NAT

정답: 3

해설: RBAC는 Role Based Access Control로 역할에 따라 권한을 부여한다.
