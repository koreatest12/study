# 어플리케이션 보안 심화정리

---

## 1. 웹 보안 핵심 구분

| 구분 | 핵심 내용 | 대표 대응 |
|---|---|---|
| SQL Injection | 입력값으로 질의 구조가 변형되는 취약점 | Prepared Statement |
| XSS | 브라우저에서 비정상 스크립트가 실행되는 취약점 | 출력값 인코딩 |
| CSRF | 인증 상태를 악용한 비정상 요청 | CSRF Token |
| 파일 업로드 취약점 | 위험 파일 업로드와 실행 가능성 | 확장자 검증, 실행 권한 제거 |
| 세션 보안 문제 | 세션 값 탈취 또는 고정 | HTTPS, HttpOnly, Secure, 세션 재발급 |

---

## 2. 입력값 검증과 출력값 인코딩

| 구분 | 설명 |
|---|---|
| 입력값 검증 | 허용된 값만 통과시키는 방식이 안전하다. |
| 출력값 인코딩 | HTML, JavaScript, URL 등 문맥에 맞게 인코딩한다. |
| 오류 메시지 제한 | 내부 경로, DB 구조, 스택 정보가 노출되지 않게 한다. |
| 권한 검증 | 화면 버튼 숨김만으로는 부족하고 서버에서도 권한을 확인해야 한다. |

---

## 3. 인증과 세션

| 항목 | 핵심 |
|---|---|
| 비밀번호 저장 | Salt와 안전한 해시 방식 사용 |
| 세션 쿠키 | HttpOnly, Secure, SameSite 설정 |
| 로그인 후 처리 | 세션 ID 재발급 |
| 로그아웃 | 서버 측 세션 무효화 |
| 중요 작업 | 재인증 또는 추가 확인 |

---

## 4. 예상문제

### 문제 1
SQL Injection의 대표적인 대응 방법은?

1. Prepared Statement
2. 관리자 권한 사용
3. 오류 상세 출력
4. 평문 쿠키 저장

정답: 1

해설: Prepared Statement는 질의 구조와 입력값을 분리한다.

### 문제 2
XSS 방어의 핵심은?

1. 출력값 인코딩
2. 포트 변경
3. 백업 삭제
4. 관리자 권한 확대

정답: 1

해설: XSS는 브라우저 실행 문맥에서 발생하므로 출력값 인코딩이 중요하다.
