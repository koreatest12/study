# 정보보안 관리 및 법규 심화정리

---

## 1. 정보보호 관리체계

| 항목 | 설명 |
|---|---|
| 정보보호 정책 | 조직의 정보보호 방향과 기준 |
| 정보보호 조직 | 책임자, 담당자, 역할과 책임 |
| 위험관리 | 자산, 위협, 취약점, 위험 평가 |
| 보호대책 | 관리적, 기술적, 물리적 보호조치 |
| 지속 개선 | PDCA 사이클 기반 개선 |

---

## 2. 위험관리

| 요소 | 설명 |
|---|---|
| 자산 | 보호할 가치가 있는 대상 |
| 위협 | 자산에 피해를 줄 수 있는 원인 |
| 취약점 | 위협이 악용할 수 있는 약점 |
| 위험 | 피해 발생 가능성과 영향 |
| 보호대책 | 위험을 줄이기 위한 조치 |

---

## 3. BCP와 DRP

| 구분 | 의미 |
|---|---|
| BCP | 업무 연속성 계획 |
| DRP | 재해 복구 계획 |
| RTO | 복구 목표 시간 |
| RPO | 복구 목표 시점 |

---

## 4. 개인정보 처리 원칙

| 원칙 | 설명 |
|---|---|
| 최소 수집 | 목적에 필요한 범위만 수집 |
| 목적 제한 | 수집 목적 범위 내에서 이용 |
| 안전성 확보 | 접근통제, 암호화, 로그 관리 등 |
| 파기 | 목적 달성 후 지체 없이 파기 |
| 정보주체 권리 | 열람, 정정, 삭제, 처리정지 요구 가능 |

---

## 5. 예상문제

### 문제 1
PDCA에서 Check 단계의 의미는?

1. 계획 수립
2. 실행
3. 점검
4. 개선

정답: 3

해설: PDCA는 Plan, Do, Check, Act 순서이며 Check는 점검 단계이다.

### 문제 2
RPO의 의미는?

1. 복구 목표 시점
2. 복구 목표 시간
3. 암호화 키
4. 방화벽 정책

정답: 1

해설: RPO는 Recovery Point Objective로 어느 시점까지 데이터를 복구할 것인지 의미한다.
